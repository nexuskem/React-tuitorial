Chapter 1

What is React?

React is a JavaScript library for building user interfaces. Created by Facebook in 2013, it is now maintained by Meta and a massive open-source community.

The Core Idea

Before React, building UIs meant manually manipulating the DOM — finding elements, changing their content, updating classes. It worked, but became a nightmare as apps grew.

React flips the model: you describe what the UI should look like at any given moment, and React figures out how to update the DOM efficiently. You stop thinking about DOM manipulation and start thinking about state and data.

// Old way: imperative (you tell the browser HOW)
const el = document.getElementById('count')
el.textContent = `Count: ${count}`

// React way: declarative (you describe WHAT it looks like)
return <p>Count: {count}</p>

Why React?

  • 🧩
    Component-basedYour UI is made of small, reusable pieces. Build once, use everywhere.
  • ⚡
    Virtual DOMReact keeps a virtual copy of the DOM and only updates what actually changed — making it fast.
  • 🌊
    Unidirectional data flowData flows down from parent to child. Predictable, easy to debug.
  • 🌎
    Massive ecosystemReact Native for mobile, Next.js for full-stack, thousands of libraries, and millions of jobs worldwide.

React vs the Alternatives

FrameworkByStyleLearning Curve
ReactMetaLibrary (bring your own router, etc.)Moderate
VueCommunityProgressive frameworkEasy
AngularGoogleFull opinionated frameworkSteep
SvelteCommunityCompiler-basedEasy-Moderate
Note: React is technically a library, not a framework. It only handles the view layer. For routing, data fetching, and state management you pull in separate tools — covered later.
Chapter 2

Setup & Your First App

Two main ways to start a React project: Vite (modern, fast, recommended) or Create React App (older, heavier). We'll use Vite.

Prerequisites

Make sure you have Node.js installed (v18 or higher). Check with:

node --version
npm --version

Create a React App with Vite

# Create new project
npm create vite@latest my-app -- --template react

# Move into the folder
cd my-app

# Install dependencies
npm install

# Start dev server
npm run dev

Open http://localhost:5173 and you'll see your React app running!

Project Structure

my-app/
├── public/          # Static files
├── src/
│   ├── App.jsx      # Your root component
│   ├── App.css      # Styles for App
│   ├── main.jsx     # Entry point — mounts React to the DOM
│   └── index.css    # Global styles
├── index.html       # HTML template
└── package.json

Understanding main.jsx

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

// Find the <div id="root"> in index.html and render your app inside it
ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)

Your First Edit

Open src/App.jsx, delete everything, and replace with:

function App() {
  return (
    <div>
      <h1>Hello, React!</h1>
      <p>My first React app</p>
    </div>
  )
}

export default App

Save it — the browser updates instantly. That's Hot Module Replacement (HMR) — no manual refresh needed.

VS Code Extensions to install: ES7+ React/Redux Snippets, Prettier, and ESLint for a smooth dev experience.
Chapter 3

JSX Deep Dive

JSX stands for JavaScript XML. It lets you write HTML-like syntax inside JavaScript. It is not HTML — it compiles to React.createElement() calls under the hood.

JSX vs HTML — Key Differences

HTMLJSXWhy
class="..."className="..."class is reserved in JS
for="..."htmlFor="..."for is reserved in JS
<br><br />All tags must be closed
onclick="fn()"onClick={fn}camelCase events, JS reference
style="color:red"style={{color: 'red'}}Styles are JS objects

Embedding JavaScript in JSX

Use curly braces {} to drop any JavaScript expression into JSX:

const name = 'Alice'
const score = 95

return (
  <div>
    <h1>Hello, {name}!</h1>
    <p>Score: {score * 2}</p>
    <p>{score > 90 ? 'Pass' : 'Retry'}</p>
  </div>
)

JSX Must Return One Root Element

// Wrong — two root elements
return (
  <h1>Title</h1>
  <p>Paragraph</p>
)

// Option 1 — wrap in a div
return (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
)

// Option 2 — Fragment (no extra DOM node)
return (
  <>
    <h1>Title</h1>
    <p>Paragraph</p>
  </>
)

Rendering Lists

Use .map() to render arrays. Always provide a unique key prop:

const fruits = ['Apple', 'Banana', 'Mango']

return (
  <ul>
    {fruits.map((fruit, index) => (
      <li key={index}>{fruit}</li>
    ))}
  </ul>
)

Conditional Rendering

// Ternary — show one of two things
{isLoggedIn ? <Dashboard /> : <Login />}

// && operator — show or show nothing
{hasError && <ErrorMessage />}
Watch out: 0 && <Component /> renders 0 on screen, not nothing. Use count > 0 && ... or !!count && ... instead.
Chapter 4

Components

Components are the building blocks of React. A component is a function that returns JSX. You compose your entire UI from components — big and small.

Function Components

// A simple component — always PascalCase name
function Greeting() {
  return <h2>Hello, World!</h2>
}

// Arrow function style — same result
const Greeting = () => <h2>Hello, World!</h2>

// Use it like an HTML tag
function App() {
  return (
    <div>
      <Greeting />
      <Greeting />
    </div>
  )
}

Composing Components

function Header() {
  return <header><h1>My App</h1></header>
}

function Footer() {
  return <footer>© 2025</footer>
}

function App() {
  return (
    <>
      <Header />
      <main>Main content</main>
      <Footer />
    </>
  )
}

File Structure Convention

src/
├── components/
│   ├── Header.jsx
│   ├── Footer.jsx
│   └── Button.jsx
├── pages/
│   ├── Home.jsx
│   └── About.jsx
└── App.jsx

Exporting Components

// Default export — one per file, import with any name
export default function Button() { ... }
import Button from './Button'

// Named export — can have multiple, must use exact name
export function PrimaryButton() { ... }
export function SecondaryButton() { ... }
import { PrimaryButton, SecondaryButton } from './Button'
Rule of thumb: One component per file. Name the file the same as the component. Use PascalCase for both.
Chapter 5

Props

Props (short for properties) are how you pass data from a parent component down to a child component. They make your components reusable and dynamic.

Passing & Receiving Props

// Parent passes props like HTML attributes
function App() {
  return (
    <Card
      title="React Hooks"
      description="Master useState and useEffect"
      rating={5}
    />
  )
}

// Child receives them as a 'props' object
function Card(props) {
  return (
    <div>
      <h2>{props.title}</h2>
      <p>{props.description}</p>
      <p>Rating: {props.rating}/5</p>
    </div>
  )
}

Destructuring Props (preferred)

// Much cleaner — destructure in the parameter
function Card({ title, description, rating }) {
  return (
    <div>
      <h2>{title}</h2>
      <p>{description}</p>
    </div>
  )
}

Default Props

function Button({ label = 'Click me', color = 'blue' }) {
  return <button style={{color}}>{label}</button>
}

<Button />              // Uses both defaults
<Button label="Submit" /> // Custom label, default color

The children Prop

Anything you put between component tags becomes the special children prop:

function Box({ children }) {
  return <div className="box">{children}</div>
}

<Box>
  <h2>I am inside Box</h2>
  <p>Any content here becomes children</p>
</Box>
Props are read-only! Never modify props inside a component. They flow down from parent to child and are immutable. To change data, you need state.
Chapter 6

State & useState

State is data that changes over time. When state changes, React re-renders the component with the new values. useState is the hook that gives components their own memory.

Your First useState

import { useState } from 'react'

function Counter() {
  // [currentValue, setterFunction] = useState(initialValue)
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
    </div>
  )
}

State with Objects

const [user, setUser] = useState({ name: 'Alice', age: 25 })

// Never mutate state directly
user.name = 'Bob'  // Will NOT trigger re-render!

// Always create a new object with spread
setUser({ ...user, name: 'Bob' })

State with Arrays

const [todos, setTodos] = useState([])

// Add item
setTodos([...todos, newTodo])

// Remove item
setTodos(todos.filter(t => t.id !== id))

// Update item
setTodos(todos.map(t => t.id === id ? { ...t, done: true } : t))

Functional Updates

When new state depends on old state, use the function form of the setter:

// Risky if called multiple times rapidly
setCount(count + 1)

// Safe — React guarantees prev is the current value
setCount(prev => prev + 1)

Lifting State Up

When sibling components need to share state, lift it up to their common parent:

function Parent() {
  const [value, setValue] = useState('')
  return (
    <>
      <Input value={value} onChange={setValue} />
      <Display value={value} />
    </>
  )
}
Key insight: State updates are asynchronous. After calling setCount(5), count is still the old value on the next line. React batches and applies all updates before the next render.
Chapter 7

Events & Forms

React uses synthetic events — a cross-browser wrapper around native DOM events. Event names are camelCase, and handlers receive an event object.

Common Event Handlers

// Click
<button onClick={handleClick}>Click</button>

// Input change
<input onChange={handleChange} />

// Form submit
<form onSubmit={handleSubmit}>...</form>

// Mouse enter / leave
<div onMouseEnter={fn} onMouseLeave={fn}>...</div>

// Key press
<input onKeyDown={e => e.key === 'Enter' && submit()} />

Controlled Components

In a controlled input, React state is the single source of truth. Every keystroke updates state, and state drives what the input shows:

function LoginForm() {
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')

  const handleSubmit = (e) => {
    e.preventDefault()  // Stop page refresh
    console.log({ email, password })
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={e => setEmail(e.target.value)}
      />
      <input
        type="password"
        value={password}
        onChange={e => setPassword(e.target.value)}
      />
      <button type="submit">Login</button>
    </form>
  )
}

Handling Multiple Fields — One Handler

const [form, setForm] = useState({ name: '', email: '', age: '' })

const handleChange = (e) => {
  const { name, value } = e.target
  setForm(prev => ({ ...prev, [name]: value }))
}

// Give each input a name matching the state key
<input name="name"  value={form.name}  onChange={handleChange} />
<input name="email" value={form.email} onChange={handleChange} />
For complex forms: consider React Hook Form or Formik. They handle validation, touched states, and errors elegantly. Understand controlled inputs first, then graduate to these.
Chapter 8

useEffect & the Lifecycle

useEffect lets you perform side effects — things that reach outside of React: API calls, subscriptions, timers, localStorage, and direct DOM access.

The Three Patterns

import { useEffect } from 'react'

// 1. Runs after EVERY render (rarely what you want)
useEffect(() => {
  console.log('rendered')
})

// 2. Runs ONCE on mount (empty dependency array)
useEffect(() => {
  console.log('mounted!')
  fetchData()
}, [])

// 3. Runs when specific values change
useEffect(() => {
  fetchUserData(userId)
}, [userId])

Fetching Data

function UserProfile({ userId }) {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    setLoading(true)
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data)
        setLoading(false)
      })
  }, [userId])

  if (loading) return <p>Loading...</p>
  return <h2>{user?.name}</h2>
}

Cleanup Function

Return a function from useEffect to clean up subscriptions or timers when the component unmounts:

useEffect(() => {
  const timer = setInterval(() => {
    setTime(t => t + 1)
  }, 1000)

  // Runs when component unmounts or before next effect
  return () => clearInterval(timer)
}, [])
Common mistake: Including a function in the dependency array that gets recreated every render causes an infinite loop. Define functions inside the effect, or wrap them with useCallback.
Modern alternative: TanStack Query (React Query) handles data fetching, caching, loading states, and errors far better than bare useEffect. Highly recommended once you understand the fundamentals.
Chapter 9

More Hooks

Beyond useState and useEffect, React ships several more hooks for refs, complex state, performance optimization, and building your own reusable logic.

useRef

Two main uses: (1) access DOM elements directly, (2) store a mutable value that persists between renders without triggering re-renders.

import { useRef, useEffect } from 'react'

// Use 1: DOM access
function AutoFocus() {
  const inputRef = useRef(null)
  useEffect(() => { inputRef.current.focus() }, [])
  return <input ref={inputRef} />
}

// Use 2: Mutable value (no re-render on change)
const renderCount = useRef(0)
renderCount.current += 1

useReducer

Like useState but for complex state logic — multiple related values, or when next state depends on previous in non-trivial ways:

import { useReducer } from 'react'

const initialState = { count: 0, step: 1 }

function reducer(state, action) {
  switch (action.type) {
    case 'increment': return { ...state, count: state.count + state.step }
    case 'decrement': return { ...state, count: state.count - state.step }
    case 'setStep':   return { ...state, step: action.payload }
    default: return state
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState)
  return (
    <>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
    </>
  )
}

useMemo & useCallback

import { useMemo, useCallback } from 'react'

// useMemo — memoize an expensive computed value
const result = useMemo(() => {
  return expensiveComputation(data)
}, [data])  // Only recalculates when data changes

// useCallback — stable function reference across renders
const handleClick = useCallback(() => {
  doSomething(id)
}, [id])  // Only recreates when id changes
Don't over-optimize: Add useMemo/useCallback only when you've measured a real performance problem. They add complexity and have their own overhead.

Custom Hooks

Build your own reusable hooks by composing existing ones. Custom hook names always start with use:

function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false))
  }, [url])

  return { data, loading, error }
}

// Using the custom hook
const { data, loading } = useFetch('/api/posts')
Chapter 10

Context API

Context solves prop drilling — passing data through many layers of components that don't need it, just to reach a deeply nested child.

The Problem (Prop Drilling)

// 'user' needs to reach UserAvatar, but has to pass through everything
<App user={user} />
  <Header user={user} />       // doesn't need user
    <Nav user={user} />         // doesn't need user
      <UserAvatar user={user} />  // finally uses user

Creating & Using Context

import { createContext, useContext, useState } from 'react'

// 1. Create the context
const UserContext = createContext(null)

// 2. Provide it — wrap the component tree that needs access
function App() {
  const [user, setUser] = useState({ name: 'Alice' })
  return (
    <UserContext.Provider value={{ user, setUser }}>
      <Header />  // No user prop needed!
    </UserContext.Provider>
  )
}

// 3. Consume anywhere inside the Provider
function UserAvatar() {
  const { user } = useContext(UserContext)
  return <span>{user.name}</span>
}

Best Practice — Custom Provider + Hook

const UserContext = createContext(null)

export function UserProvider({ children }) {
  const [user, setUser] = useState(null)
  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  )
}

// Custom hook gives better errors and cleaner imports
export function useUser() {
  const context = useContext(UserContext)
  if (!context) throw new Error('useUser must be inside UserProvider')
  return context
}
Context is not a Redux replacement. Every consumer re-renders when the context value changes. For high-frequency state (animations, large lists), use Zustand or Redux Toolkit. Context is perfect for infrequent data like theme, auth user, or locale settings.
Chapter 11

React Router

React doesn't include routing built-in. React Router v6 is the standard library for multi-page React apps with URL-based navigation — no page reloads.

Install

npm install react-router-dom

Basic Setup

import { BrowserRouter } from 'react-router-dom'

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
)

Defining Routes

import { Routes, Route } from 'react-router-dom'
import Home from './pages/Home'
import About from './pages/About'
import User from './pages/User'
import NotFound from './pages/NotFound'

function App() {
  return (
    <Routes>
      <Route path="/"         element={<Home />} />
      <Route path="/about"    element={<About />} />
      <Route path="/user/:id" element={<User />} />
      <Route path="*"          element={<NotFound />} />
    </Routes>
  )
}

Navigation

import { Link, NavLink, useNavigate } from 'react-router-dom'

// Link — basic navigation (no page reload)
<Link to="/about">About</Link>

// NavLink — adds 'active' class automatically
<NavLink to="/about" className={({ isActive }) => isActive ? 'active' : ''}>
  About
</NavLink>

// useNavigate — programmatic navigation
const navigate = useNavigate()
navigate('/dashboard')   // go to route
navigate(-1)             // go back

URL Params & Query Strings

import { useParams, useSearchParams } from 'react-router-dom'

// /user/42 → useParams gives { id: "42" }
function User() {
  const { id } = useParams()
  return <p>User ID: {id}</p>
}

// /search?q=react → get query params
const [searchParams] = useSearchParams()
const query = searchParams.get('q')
